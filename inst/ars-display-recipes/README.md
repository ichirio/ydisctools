# ARS display-recipe catalog

`recipes.json` maps a **display type** (one row of the TOC workbook, see
`ars_toc_template()`) to the compact-parameter `Analyses` rows that
`ars_from_toc()` expands and `build_ars()` turns into ARS metadata.  The same
recipe drives the mock shell generated by `ars_mock()`.

The layouts are transcribed from public standard-display definitions:

- FDA, *Standard Safety Tables and Figures: Integrated Guide*
  (<https://www.fda.gov/media/187065/download>) - display IDs referenced per
  recipe (fda-dm-t02, fda-ds-t04, fda-ex-t05, fda-ae-t06/t12/t13); the CDISC
  eTFL Portal (<https://www.cdisc.org/kb/etfl>) publishes ARS-compliant
  packages for the same displays.
- PHUSE Working Group *Standard Analyses & Code Sharing* white papers
  (*Analyses & Displays Associated with Demographics, Disposition and
  Medication* / *... with Adverse Events*), freely available from the PHUSE
  deliverables catalog.

Simplifications (deliberate, documented per recipe in `source`):

- the nested arm x SOC x PT display needs three simultaneous groupings, which
  the compact format does not support yet (ydisctools issue #6) - the
  catalog ships flat by-SOC and by-PT displays instead;
- exposure uses the ADSL duration/dose variables (`TRTDURD`, `CUMDOSE`)
  rather than the ADEXSUM parameter structure.

Recipe fields: `id`, `title_default`, `section_key` (numbering section key:
`baseline` / `efficacy` / `safety_ae` / `safety_lab`, see below), `source`,
`stub_label` (mock row-label header), and `analyses` - each with `name`,
`method` (bundled method-catalog key), `dataset`, `variable`, optional
`group2` (category variable, becomes the second ARS grouping; qualify with a
dataset as in `ADAE.AEBODSYS` when it is not on ADSL), optional `where`
(condition mini-syntax; combined with the TOC row's extra `where` via AND),
optional `denominator` (`"auto"`).

The TOC row supplies the study specifics injected at expansion time:
`population`, `group_by` (the arm variable), `groups` (pre-defined arm
levels) and extra `where` conditions.

## Numbering

`ars_from_toc()` respects an explicit `toc_no`; rows without one are
auto-numbered `<section prefix>.<sequence within section>` from the recipe
(or TOC row) `section_key` and a **section map**.  The default map follows
the ICH E3 section-14 convention (`baseline = 14.1`, `efficacy = 14.2`,
`safety_ae = 14.3.1`, `safety_lab = 14.3.4`).  Real numbering conventions
are sponsor-specific (assigned by biostatistics at TFL-spec time, usually
anchored to the company CSR template), so the map is an argument: pass your
own named vector, or derive one from a company CSR template (planned
`read_csr_map()`, ydisctools issue #18).
